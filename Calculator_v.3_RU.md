### **2. Краткое описание проекта.**

• Проект Calculator v.3 – это многофункциональное десктопное приложение на Python с графическим интерфейсом, предназначенное для выполнения широкого спектра математических операций. 

• Программа объединяет функции простого, инженерного и научного калькулятора, предоставляя инструменты для арифметических вычислений, работы с простыми числами, решения уравнений, тригонометрических преобразований, построения графиков и других математических задач в удобном и современном интерфейсе с сохранением истории работы.

• В отличие от второй версии Calculator v.3 включает в себя полноценную систему настроек с возможностью смены языка интерфейса (русский/английский) и темы оформления (тёмная/светлая) с автоматическим сохранением параметров между запусками.

### **5. Инструкция по использованию.**

• Перед использованием приложения важно понять как оно работает. Первым делом следует ознакомиться с функциями, которые находятся в левой панели экрана. При наведении курсора на кнопку появляется формула выбранной операции.

• После определения нужно нажать на нужную функцию и перейти на правую панель, где появятся строки для ввода чисел, чтобы провести выбранную операцию. 

• Чтобы было понятнее какая вводимая буква что обозначает, прямо под строками ввода есть подсказка в виде формулы с теми же переменными, что задаются пользователем.

• После ввода требуется провести вычисление, нажав на кнопку ниже. После получения результата можно выбрать новую функцию, либо стереть нынешние вычисления кнопками в нижней панели. 

• Если выбранная операция – построение графика, то при нажатии на кнопку ниже появится окно с требуемым графиком, а сама программа выведет цифру 0 как ответ. 

• Для изменения языка или темы нажмите кнопку "Настройки" в правом верхнем углу. Выбранные параметры сохранятся автоматически и восстановятся при следующем запуске.

• Все вычисления запоминаются в истории, список которой доступен в верхнем правом углу. Ее можно скопировать или очистить.

### **3. Архитектура проекта.**

• Проект использует подход **Model-View-Controller (MVC)** внутри одного главного класса Calculator.

• **Модель (Model)** – это математические функции, такие как сложение, поиск НОД, решение уравнений. Они находятся в методах класса.

• **Представление (View)** – это весь графический интерфейс: кнопки, поля ввода, панели, которые создаются с помощью tkinter.

• **Контроллер (Controller)** – это логика, которая связывает интерфейс и вычисления. Она обрабатывает нажатия кнопок, ввод данных и отображение результатов.

### **4. Структура проекта.**

• Один главный файл calculator.py содержит весь код программы.

• Файл calc_history.json создается автоматически, в нем хранится история вычислений.

• Файл settings.json создается автоматически, в нем хранятся настройки языка и темы.

• В программе есть один основной класс и больше 60 методов (функций) для разных задач.

• Каждая математическая операция вынесена в отдельный метод, это делает код понятным и удобным для изменений.

### **5. Ключевые компоненты системы.**

• Менеджер интерфейса создает все кнопки, поля ввода и панели, которые видит пользователь.

• Система ввода обрабатывает то, что пользователь печатает, и умеет понимать дроби и число pi.

• Блок математических операций содержит больше 30 функций: от сложения до решения уравнений.

• Модуль графиков рисует функции в отдельном окне с осями координат и подписями.

• Система истории запоминает последние вычисления и показывает их в специальном окне.

• Модуль настроек сохраняет и загружает параметры языка и темы в файл settings.json.

### **6. Реализация пользовательского интерфейса.**

• Интерфейс выполнен в темной цветовой гамме с фиолетовыми и бирюзовыми акцентами, доступна светлая тема.

• Левая панель с кнопками разделена на цветные категории для удобной навигации.

• Правая панель меняется в зависимости от выбранной операции – появляются нужные поля для ввода.

• При наведении на кнопку она меняет цвет и показывает формулу выбранной операции.

• Работают горячие клавиши: Escape для сброса, Ctrl+C для копирования результата.

• Окно программы автоматически открывается по центру экрана при запуске.

• **(1)** Сложность: Как полностью обновить интерфейс при смене языка или темы.

Решение: Метод refresh_ui уничтожает все виджеты и заново создает интерфейс с новыми настройками.

### **9. Текущие ограничения проекта.**

• Нельзя посчитать факториал числа больше 100, чтобы программа не зависала.

• Нельзя приближать или отдалять графики, двигать их мышкой.

• Калькулятор не умеет работать с комплексными числами, кроме показа корней уравнений.

• История хранит только 50 последних записей, нельзя искать по ним или фильтровать.

• При ошибке ввода просто показывается сообщение, но неправильное поле никак не подсвечивается.

• Нельзя ввести выражение целиком, состоящее из нескольких операций, только отдельные числа для конкретного выбранного действия.

• При смене языка или темы текущая операция и введенные данные сбрасываются.
